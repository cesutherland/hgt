## Task (verbatim from the issue body)

## Problem
The executor runs green but ships nothing. Run 29883439700 (issue #50): 32 turns,
$0.70, `permission_denials_count: 14`, **no branch, no PR**. claude-code-action's
default tool policy allows file edits + read-only git, but **not arbitrary Bash** — so
`git checkout -b`, `git push`, `gh pr create` are all denied. The agent edits locally,
can't branch/push/PR, and exits "successfully."

## Fix
Grant the specific verbs via `claude_args` in `.github/workflows/hgt-execute.yml`:

```yaml
claude_args: |
  --allowedTools "Bash(git checkout:*),Bash(git branch:*),Bash(git add:*),Bash(git commit:*),Bash(git push:*),Bash(gh pr create:*)"
```

**Scope it tight — no blanket `Bash(git:*)` / `Bash(gh:*)`.** v0 reads the raw,
unnormalized issue body, so an injection runs against whatever Bash is allowed. Grant
only the verbs the job needs. The `GITHUB_TOKEN` scope + branch protection are the real
containment; the narrow allowlist is defense-in-depth on top.

## Non-goals
- Broadening the token or permissions (contents + PR write stays).
- The silent-success guard — separate ticket.
