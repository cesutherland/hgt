## Task (verbatim from the issue body)

## Scope

Fast-follow on #9 (PR #20). When `hgt work <n>` launches its detached `hgt-issue-<n>` session, split it into two panes: **claude left, a shell right** (cwd = the worktree). Today it's a single pane running claude (`launch_session`, `cmds/work.sh:105`).

## Why

A shell alongside the agent without leaving the session — run tests, `git`, poke around — while claude holds the left pane. #9 explicitly deferred pane layout ("just session lifecycle"); this is that fast-follow.

## Behavior

- Fresh launch: `new-session -d` (claude, left) → `split-window -h -c <worktree>` (shell, right). Focus lands on claude.
- Resume (live `has-session`): reattach the existing session **untouched** — don't re-split.
- `--no-tmux`: unaffected (inline claude, no panes).

## Non-goals

- Status-line theming, tmux config, >2 panes, extra windows.
- Persisting / restoring custom layouts.

## Open decision

- Split ratio — default ~60/40 claude-favored, or 50/50? Resolve in the slice.
