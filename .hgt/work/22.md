## Context

There's no `.github/ISSUE_TEMPLATE/`. Every issue so far has been hand-rolled, and while they converge on a shape (Context/Scope → Non-goals → Acceptance criteria → Dependencies), each one re-derives it. A template makes the house style the default and cheap for others — and the future `hgt issue create` — to follow.

Surfaced reviewing PR #20.

## Scope

## Non-goals

- No YAML issue *forms*, no multiple template variants (one general slice template is enough).
- No `config.yml` / contact-links, no PR template.
- Not wiring this into a future `hgt issue create` — that's downstream.
