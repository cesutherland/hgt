## Task (verbatim from the issue body)

## Scope
Make a detached **tmux** session the default launch path for `hgt work <n>`, slotting into the launcher seam Slice 2 (#8) leaves behind. Nothing else.

Blocked by #8 — this slice swaps the launcher's inline default for tmux; the inline path stays as the `--no-tmux` fallback.

## Why
You live in tmux, and a detached session sharpens the §4 crash-recovery story: a `tmux new-session -d` running the named claude session **survives your terminal dying**, so resume becomes "reattach if it's alive, else recreate" instead of "relaunch from scratch."

## Behavior
- **Launch:** `tmux new-session -d -s hgt-issue-<n> -c <worktree> 'claude -n hgt-issue-<n> …'` — detached, named to match the issue and the claude session.
- **Attach:** if already inside tmux (`$TMUX` set) use `switch-client`; otherwise `attach-session`. Don't nest.
- **Resume:** if a `hgt-issue-<n>` session already exists, attach/switch to it instead of recreating.
- **`--no-tmux`:** fall back to the Slice 2 inline launch (no tmux session).
- **Teardown:** `hgt work rm <n>` also `tmux kill-session -t hgt-issue-<n>` (if present), alongside the worktree removal.

## Non-goals
- Window/pane layout, status-line theming, a tmux config — just session lifecycle.
- Any `hgt issue` surface.

## Exit criteria
- `hgt work <n>` launches into a detached `hgt-issue-<n>` tmux session and attaches (switch-client when `$TMUX` is set, attach otherwise).
- Re-running attaches to the live session rather than spawning a second.
- `--no-tmux` reproduces the Slice 2 inline behavior exactly.
- `hgt work rm <n>` kills the session as well as the worktree.
- Tier-1 suite covers launch / attach-vs-switch (`$TMUX` set vs unset) / resume-existing / teardown via the tmux shim.
- ADR `0003` only if the attach-vs-switch + detached-as-recovery choice warrants a record; otherwise a code comment is enough — don't manufacture an ADR for a small decision.
