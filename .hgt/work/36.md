## Task (verbatim from the issue body)

## Problem
The two names hgt generates for an issue fail in opposite directions:
- Branch (`cmds/work.sh:69`) is `issue-<n>-<full-title-slug>` — e.g.
  `issue-22-chore-add-a-github-issue-template`. Long and unwieldy.
- Session (`_session_name`, `cmds/work.sh:38`) is `hgt-issue-<n>` — just the id.
  Scannable by the machine, inscrutable to a human with three sessions open.

## Scope
Rebalance both toward "short + human-readable":
- Branch: bound the slug (truncate to N words/chars; consider dropping the
  redundant conventional-commit type word).
- Session: fold a short slug into the name so `hgt-22-issue-template` beats
  `hgt-issue-22`.

Constraint: the session id is the deterministic join key for tmux + `claude --resume`
(`work.sh:35`), chosen as bare `N` so resume needs no title lookup. Any slug must stay
re-derivable from `N` alone — or keep `N` as the stable key with the slug as a cosmetic
suffix. Call out which in the fix.

## Non-goals
- Not renaming existing branches/sessions — new work only.
- No user-configurable format knob yet.

## Acceptance criteria
- [ ] Branch slug is length-bounded, not the full title.
- [ ] Session name carries a human-readable hint while remaining a stable resume key.
