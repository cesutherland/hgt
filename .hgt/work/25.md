## Task (verbatim from the issue body)

## Context

`launch_session` (cmds/work.sh) hand-quotes the detached session's command, which tmux runs via `sh -c`:

    run tmux new-session -d -s "$name" -c "$wt" "claude -n '$name' '$prompt'"

Safe only because `$prompt` currently has no `'`. First apostrophe ("don't merge") closes the quote early → garbled command or syntax error. Shim tests log args but never run `sh -c`, so they miss it. From PR #20 review.

## Scope

POSIX-sh-safe escaping (tmux uses dash, not bash) for anything interpolated into the command string. tmux path only.

**Not `printf %q`** — it emits bash `$'…'` for control chars, which dash chokes on. Either a hand escaper (wrap in `'…'`, replace `'` → `'\''`) or export the values and let the inner shell expand them.

## Non-goals

- `--no-tmux` path (argv is already safe); the prompt text; new flags.
