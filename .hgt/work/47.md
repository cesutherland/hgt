## Task (verbatim from the issue body)

## Problem
`launch_session` runs claude as the pane's PID 1 (`cmds/work.sh:170`):
```
tmux new-session -d -s "$name" -c "$wt" "claude -n '$name' '$prompt'"
```
When claude exits for *any* reason, the pane closes → last pane → the session evaporates. The next `split-window` then fails with `can't find pane`, masking the real cause. Any startup failure becomes a **total, silent** failure — contradicting ADR 0003's "the detached session is the durable artifact."

Observed trigger: the calling shell had the wrong nvm node active; claude inherited it and bailed. Env inheritance is working as intended — the bug is that the failure was total and hid what broke.

## Fix (send-keys)
Create the session as a plain shell, then launch claude *into* it:
```
tmux new-session -d -s "$name" -c "$wt"
tmux send-keys -t "$name" "<claude cmd>" Enter
```
Decouples lifecycle: if claude dies, you land in a live shell in the worktree with its stderr on screen — session intact, `claude --resume` available, failure visible. Keep the two-pane layout (claude left, shell right via `split-window`, `select-pane -L`).
