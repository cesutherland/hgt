## Problem

`hgt work <n>` picks create-vs-resume purely on whether the worktree *directory* exists (`cmds/work.sh:222`). If the directory is gone but the feature branch still exists, it takes the **create** path and runs `git worktree add -b "$branch"` — which fails, because `-b` tries to create a branch that's already there.

Not a corner case: `hgt work rm <n>` deliberately **leaves the branch intact** (`cmds/work.sh:174`, "branch left intact"). So the sequence `hgt work N` → `hgt work rm N` → `hgt work N` is broken today.

## Fix

When the worktree is absent, branch on whether the branch already exists:

- branch missing → `git worktree add -b "$branch" "$wtpath" "$base"` (today's path — fresh create + `_seed_work_state`).
- branch exists → `git worktree add "$wtpath" "$branch"` (re-attach to the existing branch). This is a **resume**: the seed-state commit already lives on the branch, so do **not** re-seed / re-commit.
